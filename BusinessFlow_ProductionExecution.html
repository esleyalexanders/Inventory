Business Flow: Production Execution

In-house and outsource production management, material picking, QC checkpoints, and progress tracking

1. Visual Flowchart

2. Flow Chart Explanation

This production execution workflow manages both in-house manufacturing and outsourced production. For in-house work, the system automates material picking, production tracking, and quality checkpoints. For outsourced production, vendor collaboration is streamlined through digital portals. All production paths converge at final QC before goods are released to inventory.

Process Flow Steps

Step 1: Work Order Created

Trigger: Work order generated from previous planning step.

Work order contains:

  • Item to be produced (BOM, semi-finished, or component)
  • Quantity required
  • Required completion date
  • Production method (in-house or outsource)
  • Bill of Materials (BOM) with component requirements
  • Routing/process instructions

System: Work order status set to "Released for Production"

Step 2: In-House or Outsource Decision

Decision: Is production done in-house or outsourced?

Decision made during planning phase based on:

  • Production capacity availability
  • Technical capability and equipment
  • Lead time requirements
  • Cost considerations
  • Quality control requirements

Path A: In-house production (Steps 3-20)

Path B: Outsource production (Steps 21-25)

In-House Production Path

Step 3: Assign to Production Line

Action: Work order assigned to specific production line/work center.

Assignment criteria:

  • Line capability and equipment requirements
  • Current workload and capacity
  • Priority and due date
  • Operator skills required

System: Updates production schedule and notifies production supervisor

Step 4: System Generates Material Pick List

Action: System automatically creates pick list for warehouse.

Pick list contains:

  • All components/materials needed (from BOM)
  • Required quantity for each component
  • Storage locations in warehouse
  • Batch/lot numbers if applicable
  • Work order number for tracking

Automation: Pick list sent to warehouse team automatically - no manual creation

Step 5: Warehouse Picks Materials

Action: Warehouse staff picks materials per pick list.

Picking activities:

  • Navigate to storage locations
  • Verify material and quantity
  • Scan barcodes for traceability
  • Stage materials for production

System: Tracks picking progress and completion

Step 6: Materials Issued to Production

Action: Materials transferred from warehouse to production floor.

System records:

  • Material issue transaction
  • Inventory reduced for issued components
  • Work-in-progress (WIP) inventory increased
  • Issue date and time logged

Automation: Inventory automatically adjusted - no manual journal entries

Step 7: Production Operator Confirms Start

Action: Operator confirms production has started.

Operator inputs:

  • Work order number
  • Operator ID
  • Start date and time
  • Machine/equipment used

System: Work order status updated to "In Production"

Step 8: Production in Progress

Action: Manufacturing/assembly work is performed.

Tracking:

  • Operator follows process instructions
  • System tracks elapsed time
  • Optional: Real-time progress updates from operators
  • System monitors if production is on schedule

Step 9: Mid-Process QC Required Decision

Decision: Does this item require mid-process quality inspection?

Mid-process QC required for:

  • Critical components with tight tolerances
  • Multi-stage processes where early defect detection saves cost
  • Customer-specified inspection points
  • High-value items

If Yes: Proceed to QC checkpoint

If No: Continue production to completion

Step 10: QC Checkpoint Inspection

Action: Mid-process quality inspection performed.

QC activities:

  • Operator or QC inspector performs checks
  • Measures dimensions, checks tolerances
  • Visual inspection for defects
  • Records results in system

Step 11: Mid-Process QC Pass Decision

Decision: Does work-in-progress pass quality standards?

If Yes (QC Pass): Continue production

If No (QC Fail): Stop production and investigate

Step 12: Stop Production - Investigate Defect

Action: Production halted to address quality issue.

Investigation activities:

  • Identify root cause of defect
  • Check material quality
  • Verify machine settings
  • Review operator technique

System: Non-conformance report (NCR) generated

Step 13: Rework or Scrap Decision

Decision: Can defective items be reworked or must be scrapped?

If Rework:

  • Assign rework operation
  • Return to production (Step 8)
  • System tracks rework quantity and cost

If Scrap:

  • Record scrap quantity
  • Adjust WIP inventory
  • May require additional material issue to meet target quantity

Step 14: Continue Production

Action: Production proceeds to completion after QC pass.

Activities:

  • Complete remaining production steps
  • Final assembly or finishing operations
  • Packaging if required

Step 15: Production Completed

Action: Operator confirms production is finished.

Operator records:

  • Actual quantity produced (good + reject)
  • Completion date and time
  • Any production notes or issues

System: Work order status updated to "Completed - Pending Final QC"

Outsource Production Path

Step 16: Send to Vendor via Portal

Action: Work order/production request sent to outsource vendor.

Information shared:

  • Product specifications and drawings
  • Required quantity
  • Delivery deadline
  • Quality requirements
  • Purchase order (PO) reference

Transmission method: Vendor portal, API, or email

Step 17: Vendor Confirms Receipt

Action: Vendor acknowledges work order and confirms acceptance.

Vendor confirms:

  • Can meet specifications
  • Delivery date is feasible
  • Production will commence

System: Tracks vendor acknowledgment and expected delivery date

Step 18: Vendor Production in Progress

Action: Vendor performs production at their facility.

Tracking:

  • Optional: Vendor provides progress updates via portal
  • System monitors if vendor is on schedule
  • Alerts sent if delays expected

Step 19: Vendor Ships Completed Goods

Action: Vendor ships finished goods to company warehouse.

Shipping documentation:

  • Packing list with quantities
  • Shipping waybill/tracking number
  • Quality certificates if required
  • Delivery note

System: Tracks shipment status

Step 20: Goods Received at Warehouse

Action: Outsourced production received at receiving dock.

Receiving activities:

  • Unload and count quantity
  • Match with work order/PO
  • Check for shipping damage
  • Log receipt in system

Final QC and Inventory Update (Both Paths Converge)

Step 21: Final QC Inspection

Action: Final quality inspection before accepting to inventory.

QC inspection:

  • Complete dimensional and functional checks
  • Visual inspection for defects
  • Compliance with specifications
  • Sample testing if required
  • Documentation review

System: QC inspector records results with pass/fail decision

Step 22: Final QC Pass Decision

Decision: Does finished product pass final quality inspection?

If Yes (QC Pass): Accept to finished goods inventory

If No (QC Fail): Reject and take corrective action

Step 23: Update Finished Goods Inventory

Action: System updates inventory with approved finished goods.

Automated inventory updates:

  • Finished goods quantity increased
  • Work-in-progress (WIP) inventory decreased (for in-house)
  • Stock status set to "Available"
  • Location assigned in warehouse
  • Batch/lot number recorded for traceability

System: Work order closed with actual costs recorded

Step 24: Reject & Notify Vendor (Exception Path)

Action: Quality rejection for outsourced production.

System actions:

  • Stock blocked from use
  • Non-conformance report (NCR) generated
  • Vendor notified automatically with defect details
  • Vendor required to provide replacement or resolution

Step 25: Goods Available for Shipping

Action: Finished goods are ready for customer orders.

System updates:

  • Inventory visible and allocatable to orders
  • Production plan updated
  • Sales team notified goods are ready

Step 26: Trigger Shipping/Invoicing

Action: System automatically triggers next workflow.

If goods produced for specific customer order:

  • Shipping workflow initiated
  • Pick/pack operations scheduled
  • Customer notified goods ready

If goods produced to stock:

  • Available for future order allocation
  • Stock levels updated for planning

Key Automation Points

✓ Automatic Material Pick List Generation

System creates complete pick lists from BOM automatically - warehouse receives exact materials needed with locations, no manual list creation.

✓ Real-Time Inventory Updates

Material issue and finished goods receipt update inventory automatically - WIP, raw materials, and finished goods always accurate in real-time.

✓ Digital Vendor Collaboration

Outsource work orders sent digitally via portal or API. Vendor progress tracked automatically reducing manual follow-up.

✓ Integrated QC Checkpoints

Quality inspections triggered automatically at defined checkpoints. QC results recorded in system with full traceability.

✓ Automated Work Order Closure

When production completes and passes QC, work order closes automatically with actual costs and quantities recorded - no manual closure process.

✓ Exception-Based Production Management

Production flows automatically from start to finish. Management only intervenes for QC failures or delays, not for routine tracking.
